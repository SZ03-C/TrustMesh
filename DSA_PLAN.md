# DSA & Interview Prep Plan — Cybersecurity Internships

> **Target:** Cybersecurity analyst / security engineer / SDE (security-focused) internships  
> **Timeline:** 8 weeks  
> **Daily commitment:** 1–2 hours

---

## Week 1–2: Arrays, Strings, Hashing (Foundation)

| Concept | LeetCode Problems |
|---|---|
| Arrays | Two Sum, Best Time to Buy/Sell Stock, Product of Array Except Self |
| Strings | Valid Anagram, Longest Substring Without Repeating Characters |
| Hashing | Group Anagrams, Top K Frequent Elements, Valid Sudoku |
| Two Pointers | Valid Palindrome, Container With Most Water, 3Sum |

**Goal:** Solve 2–3 problems daily. Focus on brute force → optimal patterns.

---

## Week 3–4: Linked Lists, Stacks, Queues, Recursion

| Concept | LeetCode Problems |
|---|---|
| Linked Lists | Reverse Linked List, Merge Two Sorted Lists, Linked List Cycle |
| Stacks | Valid Parentheses, Min Stack, Daily Temperatures |
| Queues | Implement Queue using Stacks, Sliding Window Maximum |
| Recursion | Fibonacci, Pow(x,n), Subsets, Permutations |

**Goal:** Understand pointer manipulation and LIFO/FIFO patterns.

---

## Week 5: Trees & Graphs

| Concept | LeetCode Problems |
|---|---|
| Binary Trees | Invert Tree, Max Depth, Diameter of Binary Tree, Level Order |
| BST | Validate BST, Lowest Common Ancestor, Kth Smallest |
| Graphs | Number of Islands, Clone Graph, Course Schedule |
| DFS/BFS | Word Search, Pacific Atlantic Water Flow |

**Goal:** This is critical — security often involves graph-based problems.

---

## Week 6: Sorting, Searching, Dynamic Programming

| Concept | LeetCode Problems |
|---|---|
| Binary Search | Binary Search, Find Peak Element, Search in Rotated Array |
| Sorting | Merge Sort, Quick Sort, Kth Largest Element |
| DP Basics | Climbing Stairs, House Robber, Coin Change, Longest Increasing Subsequence |
| Intervals | Merge Intervals, Non-overlapping Intervals |

**Goal:** DP is less common in cybersecurity interviews but appears in SDE roles.

---

## Week 7: Cybersecurity-Specific Interview Prep

| Topic | What to Study |
|---|---|
| Network Security | OWASP Top 10, TLS handshake, DNS, firewalls, IDS/IPS |
| Cryptography | Symmetric vs asymmetric, hashing, digital signatures, TLS |
| Web Security | XSS, CSRF, SQLi, SSRF, authentication flaws |
| Tools | Nmap, Wireshark, Burp Suite, Metasploit (be ready to discuss) |
| System Design (lite) | Design a logging system, SIEM architecture, auth flow |

**Practice explaining TrustMesh** — your project is directly relevant here.

---

## Week 8: Mock Interviews & Revision

- Revise all 40+ problems solved
- 3 full mock interviews (use peer groups or platforms)
- Polish TrustMesh demo walkthrough (2-minute pitch)
- Update LinkedIn, resume, GitHub profile

---

## Weekly Routine

```
Mon-Fri:
  Morning (30min): 1 LeetCode problem (timed, 25 min)
  Evening (45min): Review solution, study patterns, read editorial
  Night (15min): Security concept flashcards

Weekend:
  Revise weak areas
  Work on TrustMesh / side project
  Mock interview
```
