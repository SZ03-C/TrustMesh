# Placement Strategy — Cybersecurity Internships (4th Year)

---

## Resume Tips

### What to include (in order)

1. **TrustMesh project** — Lead with this. Describe it as: *"Cybersecurity trust passport that evaluates org security posture across domain, email, exposure, and developer trust signals. Generates a 0-100 trust score with actionable remediation."*
2. **Network-Security-Playground** — *"Built hands-on labs for Nmap, Wireshark, MITM simulation, and network enumeration."*
3. **Cybersecurity-Fundamentals** — *"Authored a structured 8-month curriculum covering offensive security, defensive security, compliance, and cryptography."*
4. **Blockchain-Developer-Roadmap** — *"Created a 20-week learning path for blockchain development including Solidity, smart contract security, and DApp integration."*
5. **Skills** — Network security, OWASP Top 10, Nmap, Wireshark, Burp Suite, JavaScript, SQL, Git, Linux, OSINT

### Format

- **One page** max
- Use a clean ATS-friendly template (JSON Resume, Jake's Resume, or Overleaf)
- Include GitHub + LinkedIn links prominently
- Add a "Security Clearance / CTF" section if applicable

## TrustMesh as Your Portfolio Centerpiece

TrustMesh is your strongest asset — here's how to use it:

### In interviews
> *"I built TrustMesh because I noticed that organizations struggle to make quick trust decisions about vendors. Most scanners produce raw technical data — TrustMesh turns that into a clear trust score with business-readable decisions. It evaluates domain security, email security (SPF/DKIM/DMARC), public exposure, and developer hygiene on GitHub."*

### Demo walkthrough (30 seconds)
1. Type a domain → passport generates in ~3 seconds
2. Show the score ring animation
3. Point to the Top Risks section
4. Highlight the Collaboration Decision
5. Mention the evidence timeline

### Next features you can discuss
- Real scanner integration (DNS, TLS, security headers)
- PostgreSQL backend for scan history
- PDF report export
- Org-to-org trust request workflow
